### Q2
All five env vars are documented in `kb/runbooks/test-environment.md`. Each has zero production effect when unset; each is observability/instrumentation only. NF4 (state-confinement) is preserved because none introduces writes outside allowed paths; NF5 (secrets) is preserved because none touches the env-var scrubbing surface — they only inject deterministic behavior into stubs or observe writes.

### Q3
`EUNSTABLE` fires, not `EFORMAT`. The two-stage stability check (structural then semantic) is the resolution: the parser treats zero-byte input as a special case so the missing-required-sections check fires first, before frontmatter strictness would otherwise flag `EFORMAT`. Documented in `kb/properties/edge-cases.md#T1`'s implementation note (added in Phase 6 KB sync) and consistent with `kb/spec/algorithms.md#stability`.
